Arduino based obstacle avoidance robot

The following content will be related to his final year project

Basic Idea

The base goal of this project is to design, build and program a small wheeled robot.
This robot is to independantly move around freely within its environment and avoid the various obstacles it may face

Background

I tend to experiment with electronic components on a regular basis. Due to this I try and make small working systems, originaly nothing more than a small timer or an audio amplifier.
Naturaly the progression from pure electronics was to move onto microcontrollers given my computer science background.
After making various projects using pic and arduino microcontrollers I decided to make an Arduino based robot.
This robot was just a chassis kit (magician), which included the frame, two wheels, two motors and a battery holder. From this all I had to add was an Arduino microcontroller and a sensor, for this I used a sharpe IR (infra-red) sensor, mounted inside the shassis pointing out of the front.
The code for this early 'prototype' was very simplistic and was along these sorts of lines: 

void loop(){
if(sensor_range < value)
	{
		motors.turn(random);
	}else
	{
		motors.forward(1);
	}
}

This obviously did not work very well, but it did demonstrate a point. That point was mainly just the fact that avoiding things that are only directly in front of you is a bad plan.
If something approaches diagonaly it can avoid the single sensor completely, and this is all asuming either the robot is actually moving in a straight line or everything in the environment is stationary.

The coding was then changed so that still just using the single front facing sensor, it could (very) roughly tell which direction (left or right) had the biggest empty space to move into.
This was done by having the robot 'wiggle', it turned small amounts left and right, taking sensor readings in each direction, as well as forwards. After doing so, if there was something in range of the sensor straight ahead it would turn in the direction of the next biggest reading, and continued on in the same manner.

This proved to be a significant improvement on the previous method but still the robot found its way into objects very quickly. At this time I considered either adding more sensors or putting the existing sensor onto some servos, and scan the environment before attempting to move anywhere, this would be a more active method or determining a direction in which to move, unlike the mainly reactive method employed previous.
